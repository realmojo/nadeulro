## 1. 한 줄 소개
**액티브 시니어(50~60대, '신청년')를 위한 여가·나들이 코스 플랫폼.**
파크골프 · 온천 · 수영 · 등산 · 숙박을 하나의 "하루 코스"로 잇는다.
태그라인 예시: **"파크골프·온천·등산·숙박까지, 오늘 뭐하지?"**
---
## 2. 타깃 사용자
- 50대 이상. 스스로를 '노인'이 아니라 트렌드 키워드 그대로 **'신청년(新靑年)'**으로 인식.
- 여가·문화 소비에 지갑을 여는 층. 여행·맛집·온천 소비가 활발.
- 파크골프 기준 남성 비중이 높음(약 7:3), 50~60대가 이용자의 약 70%.
- 네이버 검색·블로그 의존도가 높고, 최근 AI 비서로 국내 여행을 기획하기 시작.
- **UX 원칙:** 큰 글씨, 높은 명도 대비, 큰 탭 타깃, 단순한 내비게이션, 모바일 우선.
- **금지 표현:** "시니어 / 실버 / 노인" — 주 고객을 밀어냄. 대신 '어른', '신청년', '나들이' 톤.

## 3. 카테고리 구성과 각자의 역할
| 카테고리 | 역할 | 메모 |
|---|---|---|
| 파크골프 | **간판(SEO 유입구)** | 뾰족한 검색 키워드, "어디 있지?" 발견 수요 큼, 장비 제휴 유리 |
| 온천 | **수익 엔진** | 여행 제휴(숙박·입장권·패키지) 단가가 가장 높음, 발견 수요도 있음 |
| 숙박 | 코스 완성 + 예약 제휴 | 온천·여행과 묶어 1박 코스로 |
| 등산 | 자연·건강 콘텐츠 축 | 코스 연결에 활용 |
| 수영 | 겨울/실내 보조 콘텐츠 | 발견보다 '강습·자유수영 시간표'가 핵심(심화 단계) |
**연결 축(차별화 핵심):** 개별 디렉터리를 나열하지 말고 하나의 코스로 꿴다.
예) `○○ 파크골프장에서 18홀 → 차로 15분, △△온천에서 휴식 → 근처 □□ 맛집/숙박`

## 4. 수익 모델
- **광고 + 제휴(affiliate) 중심.**
- 제휴 우선순위: **여행·숙박(고단가) > 파크골프 장비 > 기타.**
  - 파크골프 입문 세트 8~15만 원대 → 장비 제휴 여력 있음.
  - 수영복·수경 등은 단가 낮음 → 제휴 약함.
- 광고 슬롯은 콘텐츠 흐름을 해치지 않는 위치에.

## 7. 첫 버전(MVP) 기능 범위
1. **파크골프장 찾기** — 지도 + 필터(지역/거리/홀수) + 상세(위치·요금·길찾기).
2. **입문 가이드 콘텐츠** — 파크골프란 / 비용 / 장비 / 규칙 (SEO 유입).
3. **코스 묶음 페이지** — 파크골프 + 온천 + 맛집/숙박 하루·1박 코스 (**차별화 핵심**).
4. **온천·숙박 제휴 링크** — 수익화 시작점.
5. **FAQ(질문형) 섹션** — AEO 대응.
---
## 8. SEO / AEO / GEO 전략
**공통 뼈대: "질문형 제목 + 즉답 + 구조화 데이터 + 코스 콘텐츠".**
- **SEO (네이버 1순위):**
  - 활동×지역 롱테일 랜딩페이지("○○ 파크골프장", "△△ 근처 온천", "부모님 온천 숙박 추천").
  - 코스 콘텐츠를 내부링크 허브로.
  - JSON-LD: `LocalBusiness`, `TouristAttraction`, `FAQPage`.
- **AEO (음성·답변엔진):**
  - 사람들이 말로 묻는 문장을 그대로 H2/FAQ 제목으로.
  - 질문 바로 밑에 40~60자 **직답 먼저**, 상세는 그다음.
  - 발음 쉬운 브랜드명이 유리(나들로 = 짧고 명확).
- **GEO (ChatGPT·Perplexity·AI 개요):**
  - 사실·수치·정리된 리스트·최신 날짜 위주 서술.
  - "액티브 시니어 여가 = 나들로" 연상되게 브랜드 일관 반복.
  - 보도자료·디렉토리·위키 등재로 신뢰 신호 축적.

## 9. 접근성 (50+ 필수)
- 기본 폰트 18px 이상, 본문 대비비 충분히(WCAG AA 이상).
- 큰 버튼/링크(탭 타깃 44px+), 단순한 1~2단계 내비게이션.
- 모바일 우선, 키보드 포커스 표시, 모션 최소·`prefers-reduced-motion` 존중.

## 10. 기술 스택 제안 (Claude Code용)
- **프레임워크:** Next.js(React) — SSR/SSG로 SEO·GEO에 유리. *콘텐츠 비중이 크면 Astro도 강력한 대안.*
- **지도:** 카카오맵 JavaScript SDK.
- **데이터 계층:** 초기엔 공공데이터 CSV → 정적 JSON/SQLite. 확장 시 **Supabase(Postgres)**.
- **구조화 데이터:** `next-seo` 또는 수동 JSON-LD 삽입.
- **배포:** Vercel 또는 Cloudflare Pages.
- **환경변수(.env):** 카카오 JS 앱키, (있으면) 공공데이터 서비스키, 제휴 파트너 키.

## 12. 빌드 로드맵 (단계)
- **P0 — 기반:** 리포 세팅, 프레임워크, 랜딩+태그라인, 접근성 디자인 토큰, 카카오 앱키 연결.

- **P2 — 콘텐츠/SEO:** 입문 가이드 + FAQ + JSON-LD 구조화 데이터.
- **P3 — 코스 & 수익:** 코스 묶음 페이지 + 온천·숙박 제휴 링크.
- **P4 — 확장:** 수영(시간표)·등산 콘텐츠, GEO(보도자료·디렉토리 등재).
